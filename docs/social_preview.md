# GitHub Social Preview Guidelines

This document outlines the design specification for the `boc-allocation-review-agent` repository's GitHub social preview image. These guidelines ensure a consistent, professional, and visually compelling representation when the repository is shared on social media and portfolio platforms.

---

## 🎨 Design Specifications

### 📏 Dimensions & Format
- **Recommended Size**: 1280 × 640 pixels (exactly 2:1 aspect ratio, optimized for GitHub).
- **Safe Area**: Keep all text, logos, and critical visual elements within the central 1024 × 512 pixels to prevent clipping when shared across different platforms.
- **Export Format**: PNG (preferred for clean text rendering) or high-quality WebP.

### 🎨 Color Palette
- **Primary Background**: Sleek dark slate (`#0F172A` - Slate 900) or deep charcoal (`#0B0F19`) to give a modern, premium developer vibe.
- **Accent Glow**: Subtle cyan/blue (`#06B6D4` - Cyan 500) and electric indigo (`#6366F1` - Indigo 500) to represent agentic intelligence and tracing pathways.
- **Text & High Contrast**: Off-white (`#F8FAFC` - Slate 50) and muted text (`#94A3B8` - Slate 400).

### ✍️ Typography
- **Title Font**: A clean, modern sans-serif like **Inter**, **Outfit**, or **SF Pro Display** (semibold or bold weight).
- **Subtitle/Code Font**: **JetBrains Mono** or **Fira Code** for technical context, illustrating the developer-first nature of the tool.

---

## 🖼️ Branding & Visual Elements

### Logo & Title Placement
- **Left Column**:
  - The project title: `BOC Allocation Review Agent` in large white bold typography.
  - A descriptive subtitle: `An offline-first review co-pilot for production accounting, inspired by Google ADK.`
  - Visual badges indicating `Pure Python`, `Streamlit Dashboard`, and `Offline TF-IDF RAG`.
- **Right Column**:
  - A stylized graphic representing the decoupled agent loop.
  - Minimalistic diagram nodes labeled `Planner`, `Executor`, and `Tool Registry` with glowing connections.

---

## ⚠️ Compliance & Safety Constraints
- **Offline Representation Only**: The graphic must present the tool as a local-first review workflow.
- **No Production Overclaims**: Do not include branding elements that imply live deployment, production tax filing, or official CRA/CAVCO certification.
